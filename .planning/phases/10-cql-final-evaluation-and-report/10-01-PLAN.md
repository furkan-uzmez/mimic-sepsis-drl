<TaskGroup name="B: Sweep Orchestration Script">
Single script that runs everything: baselines → CQL sweep → evaluation → figures.

<task type="auto">
  <name>T3: Create CQL multi-seed sweep orchestrator (2-stage protocol)</name>
  <files>scripts/run_cql_sweep.py</files>
  <action>
Create `scripts/run_cql_sweep.py` implementing Tang & Wiens 2021 two-stage selection:

1. **Sparse dataset check** — verify `data/replay_sparse/replay_train.parquet` exists.

2. **Baselines** (Stage 1 only): clinician, no-treatment, behavior cloning.

3. **Stage 1 — Broad screen** (24 runs):
   - 1 seed × 2 rewards (shaped/sparse) × 3 lr (1e-4, 3e-4, 1e-3) × 4 alpha (0.05, 0.1, 0.5, 1.0)
   - All combinations screened with single seed (42)
   - Output: `runs/cql_sweep/stage1_manifest.json`

4. **Stage 2 — Multi-seed confirmation** (24 runs):
   - Reads top 6 configs from `--stage1-eval` JSON (FQE-ranked)
   - Adds 4 extra seeds (123, 456, 789, 1024) per config
   - Output: `runs/cql_sweep/stage2_manifest.json`

5. **Config overrides per run:**
   - `runtime.seed`, `extra.lr`, `extra.cql_alpha`, `extra.reward_variant`
   - `dataset_path` (shaped→data/replay/, sparse→data/replay_sparse/)
   - `logging.experiment_name`, log_dir, checkpoint_dir

CLI: `--stage 1|2 --rewards shaped,sparse --learning-rates 1e-4,3e-4,1e-3 --cql-alphas 0.1,0.5,1.0 --stage1-eval <path>`

3. **Progress tracking:** Print per-run status, elapsed time per run, total elapsed.

4. **Error handling:** If a run fails, log the error and continue with remaining runs. At the end, report which seeds completed and which failed.

CLI: `uv run python scripts/run_cql_sweep.py [--seeds 42,123,456,789,1024] [--rewards shaped,sparse]`
  </action>
  <verify>python scripts/run_cql_sweep.py --help prints usage</verify>
  <done>Single command runs all 10 CQL experiments + 3 baselines with progress tracking and error resilience.</done>
</task>

</TaskGroup>

<TaskGroup name="C: Evaluation & Diagnostics Script">
After all runs complete, compute bootstrap CIs, support diagnostics, clinician agreement.

<task type="auto">
  <name>T4: Evaluate all CQL checkpoints with bootstrap CIs and diagnostics</name>
  <files>scripts/evaluate_cql_sweep.py</files>
  <action>
Create `scripts/evaluate_cql_sweep.py` that:

1. **Load results:** Read sweep manifest from T3 output.
2. **For each (seed, reward) pair:**
   - Load final checkpoint via `load_cql_policy()`
   - Load held-out test episodes
   - Compute FQE + WIS via existing `evaluate_policy_run()`
   - Compute bootstrap CIs via new `bootstrap.py`
   - Compute support diagnostics via `safety.py`
   - Compute clinician agreement (exact + adjacent bin)
3. **Aggregate across seeds:**
   - Mean ± 95% CI of FQE per reward variant
   - Mean ± 95% CI of WIS per reward variant
   - Mean low-support action rate
   - Mean clinician agreement rate
4. **Baselines:** Evaluate clinician/no-treatment/BC with same metrics.
5. **Output:** `runs/cql_sweep/evaluation_summary.json` with all metrics.

CLI: `uv run python scripts/evaluate_cql_sweep.py`
  </action>
  <verify>python scripts/evaluate_cql_sweep.py --help prints usage</verify>
  <done>All CQL checkpoints evaluated with FQE, WIS, bootstrap CIs, support diagnostics, and clinician agreement.</done>
</task>
